# 이메일 템플릿 생성기 (UI 프로그램) 기획안

기존의 수동/AI 개입 워크플로우를 발전시켜, **사용자가 직접 조작할 수 있는 '웹 기반 로컬 UI 프로그램(HTML 파일)'**을 개발하는 기획안입니다. 사용자는 이 프로그램(웹 페이지)에 접속하거나 로컬 HTML 파일을 열어 클릭 몇 번으로 전체 과정을 자동화할 수 있습니다.

## 1. 프로그램 형태 및 기술 스택
- **형태:** 단일 로컬 HTML 파일 (`Email_Template_Builder.html`)
- **기술 스택:** HTML5, CSS (모던 UI 적용), Vanilla Javascript
- **핵심 라이브러리:** `OpenCV.js` (웹 브라우저 상에서 이미지 템플릿 매칭 수행)

## 2. 주요 기능 및 UI 흐름 (Workflow)

**Step 1: 설정 및 파일 업로드 (UI 입력부)**
- **Github 토큰 입력:** Github 자동 Push를 위한 Personal Access Token (브라우저 로컬 스토리지에 자동 저장).
- **파일명 입력:** 저장될 기준 파일명 (예: email_260601)
- **포맷 선택:** `모바일 용도(반응형)` vs `이메일 용도(고정폭 이미지 맵)` 라디오 버튼 선택.
- **이미지 업로드:** '원본 전체 이미지'와 '버튼 전용 이미지'를 각각 업로드.
- **링크 입력:** 버튼 클릭 시 이동할 타겟 URL 입력.

**Step 2: 자동 연산 처리 (내부 로직)**
- 업로드된 원본 이미지를 브라우저 내부에서 **가로 1000px로 리사이징** 처리.
- `OpenCV.js`의 템플릿 매칭 기능(`cv.matchTemplate`)을 가동하여 원본 이미지 내에서 버튼 이미지의 정확한 X, Y 좌표 자동 추출.
- 원본에서 찾은 좌표를 1000px 기준으로 자동 Scale 계산.

**Step 3: HTML 자동 생성 및 미리보기**
- 선택된 포맷(모바일/이메일)에 맞추어 이메일용 HTML 코드 자동 생성.
- 화면에 생성된 HTML 코드와 함께 매칭된 버튼 영역을 시각적으로 확인(Preview).

**Step 4: Github 자동 배포 (Push) 및 URL 추출**
- 사용자가 [Github에 배포하기] 버튼을 누르면, Github REST API를 호출하여 `hyo_email` 저장소의 `main` 브랜치에 리사이징된 이미지와 생성된 HTML 파일을 자동으로 Commit & Push.
- 성공적으로 Push되면, Cloudflare Pages 베이스 URL(`https://hyo-email.pages.dev/파일명.html`) 제공.
